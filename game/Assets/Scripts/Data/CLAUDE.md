# Scripts/Data/ — ScriptableObject Data & Config

> See root `/CLAUDE.md` for project-wide rules.

All tunable values live in ScriptableObjects. **No magic numbers in logic code.** If you need a constant that could change during balancing, it belongs in a ScriptableObject here.

## Files

| File | Class | Loaded By | Purpose |
|------|-------|-----------|---------|
| `CombatConfig.cs` | `CombatConfig` | AbilityResolver, CardHandManager | All combat math: energy, crit, mitigation, tile damage, faction bonuses |
| `EmpireConfig.cs` | `EmpireConfig` | ResourceManager, BuildingManager | Starting resources, vault caps, queue slots, offline earnings cap |
| `ProgressionConfig.cs` | `ProgressionConfig` | HeroRoster, CombatHeroFactory | XP curves, stat growth per level, max hero level |
| `HeroData.cs` | `HeroData` | HeroRoster, CombatHeroFactory | Base stats, ability pool, portrait, shard unlock cost |
| `AbilityCardData.cs` | `AbilityCardData` | CardHandManager, AbilityResolver | Cost, effect value, status, AoE, combo, visuals |
| `PassiveAbilityData.cs` | `PassiveAbilityData` | CombatHeroFactory | Always-active passive bonuses applied at battle start |
| `PveLevelData.cs` | `PveLevelData` | PveEncounterManager | Enemy lineup, terrain presets, difficulty, rewards |
| `BuildingData.cs` | `BuildingData` | BuildingManager | Costs, timers per tier, production rates |
| `ResearchNodeData.cs` | `ResearchNodeData` | ResearchManager | Prerequisites, timer, stat bonuses unlocked |
| `TerritoryConfig.cs` | `TerritoryConfig` | TerritorySystem | Territory grid size, control thresholds, alliance perks |
| `QuestConfig.cs` | `QuestConfig` | QuestEngine | Quest definitions: type, required count, cadence, rewards |

## Rules

- **Read-only at runtime.** Never write to a ScriptableObject field during play.
- **No cross-references between config SOs.** CombatConfig must not reference EmpireConfig.
- **HeroData never appears in gacha.** Only via HeroShardSystem. This is a hard design rule.
- **BuildingData timers must respect caps in BuildingManager** (4h normal / 8h Stronghold). Setting a timer here doesn't bypass the cap — BuildingManager enforces it.

## Adding a New Config Value

1. Add the field to the appropriate SO class (e.g., `CombatConfig.cs`)
2. Set a sensible default value using `[SerializeField]` + default in `OnValidate()` if needed
3. Update the asset in `Assets/Resources/` via the Inspector
4. Access it in logic code via the system that owns that config

## Adding a New ScriptableObject Type

1. Define the class in `Data/` inheriting `ScriptableObject`
2. Add `[CreateAssetMenu(menuName = "Ashen Throne/...")]` attribute
3. If it needs bulk generation, add a case to `StarterAssetGenerator.cs` or create a new generator
4. Add a row to this file's table above
